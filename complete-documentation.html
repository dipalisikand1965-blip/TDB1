useMiraFilter.js — How It Works

Client-side product ranking at frontend/src/hooks/useMiraFilter.js

  • Allergy blocking: Products with allergens matching pet's allergies are deprioritized
  • Breed matching: Products tagged for pet's breed get +2 miraRank boost
  • Size matching: Products for pet's size category get +1 boost
  • Life-stage matching: Products for puppy/adult/senior get +1 boost
  • Favorites: Pet's favorite foods matched against ALLERGEN_MAP synonyms

Critical Rules for Next Agent

1. Desktop files are LOCKED
Never modify *SoulPage.jsx files. They are production-ready and stable.
2. Never modify server.py directly
It's 24,000+ lines. Only add new routes in separate route files (unified_product_box.py, service_box_routes.py, etc.) and import them in server.py.
3. PyMongo collection truthiness check
if collection is not None: — NEVER if collection: (throws NotImplementedError)
4. Soul Products are separate from regular products
Soul products live in breed_products. They only appear on pillar pages if duplicated to products_master via "Add to catalog". This is intentional.
5. MongoDB ObjectId serialization
Always exclude _id from MongoDB responses. Use {"_id": 0} projection or strip before returning.
6. AI Image Generation requires adminAuth
Pass localStorage.getItem('adminAuth') as Authorization header. Use credentials: 'omit' to avoid browser Basic Auth popup.
7. Hot Reload
Supervisor restart only needed for .env changes or new package installs. Code changes auto-reload.
