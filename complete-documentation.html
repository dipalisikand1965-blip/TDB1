🏗️ PAGE CMS ARCHITECTURE - Standard for ALL Pillar Pages

⚠️ CRITICAL FOR ALL FUTURE AGENTS: This is the TEMPLATE for building pillar pages. Every pillar (Learn, Advisory, Care, Fit, Stay, Travel, Dine, etc.) MUST follow this architecture.

The Problem We Solved:

Previously, each pillar page was hardcoded with trial-and-error product/service assignments. This led to inconsistencies, broken modals, and endless back-and-forth fixes.

The Solution: Page CMS

A centralized Admin interface where EVERY component of a page is configurable:

Page CMS Structure (Apply to ALL Pillars)


PILLAR PAGE CMS
├── 🎨 PAGE SETTINGS
│   ├── Page Title (editable)
│   ├── Page Subtitle (editable)
│   ├── Hero Image (uploadable via Cloudinary)
│   ├── Theme Color (configurable)
│   └── Section Visibility Toggles
│
├── 📚 TOPICS SECTION
│   ├── Section Title (editable)
│   ├── Show/Hide toggle
│   └── Topics (add/remove/reorder):
│       └── EACH TOPIC has:
│           ├── Title
│           ├── Slug (URL-friendly)
│           ├── Image (uploadable)
│           ├── Description
│           └── MODAL CONTENT:
│               ├── Subtopics (add/edit/remove) → Overview Tab
│               ├── Videos (YouTube URLs) → Videos Tab
│               ├── Products (pick from catalog) → Products Tab
│               └── Services (pick from catalog) → Services Tab
│
├── 🎁 BUNDLES SECTION
│   ├── Section Title (editable)
│   ├── Show/Hide toggle
│   ├── Max items to display
│   └── Selected Bundles (pick from catalog)
│
├── 🛍️ PRODUCTS SECTION
│   ├── Section Title (editable)
│   ├── Show/Hide toggle
│   ├── Layout configuration
│   └── Featured Products (pick from catalog)
│
├── 🔧 SERVICES SECTION
│   ├── Section Title (editable)
│   └── Featured Services (pick from catalog)
│
└── 👤 PERSONALIZATION SECTION
    ├── Show breed-specific products (toggle)
    ├── Show archetype recommendations (toggle)
    └── Show "Recommended for {pet}" (toggle)

Database Collections Required:

Collection Purpose Key Fields
page_configs Page-level settings pillar, title, subtitle, heroImage, themeColor, sections
learn_topics (or {pillar}_topics) Topics for that pillar title, slug, image, description, subtopics[], videos[], products[], services[], order
page_selections Featured items per page pillar, bundles[], products[], services[]

API Endpoints Required:

Endpoint Method Purpose
/api/{pillar}/page-config GET Load entire page configuration
/api/{pillar}/page-config POST Save entire page configuration
/api/{pillar}/topic-products/{slug} GET Get products for a specific topic modal

Frontend Components Required:

Component Location Purpose
{Pillar}PageCMS.jsx /frontend/src/components/admin/ Admin CMS for that pillar page
{Pillar}Page.jsx /frontend/src/pages/ The actual pillar page (reads from CMS config)
{Pillar}TopicModal.jsx /frontend/src/components/{pillar}/ Modal that opens when user clicks a topic

Step-by-Step: Adding CMS to a New Pillar

Step 1: Create Backend Endpoints


# In {pillar}_routes.py

@router.get("/page-config")
async def get_page_config():
    db = get_db()
    config = await db.page_configs.find_one({"pillar": "{pillar}"})
    topics = await db.{pillar}_topics.find({}).sort("order", 1).to_list(50)
    selections = await db.page_selections.find_one({"pillar": "{pillar}"})
    return {"config": config, "topics": topics, "selections": selections}

const [pageConfig, setPageConfig] = useState(null);

useEffect(() => {
    fetch(`${API_URL}/api/{pillar}/page-config`)

        .then(data => {
            setPageConfig(data.config);
            setTopics(data.topics);
        });
}, []);

// Then render sections based on pageConfig.sections.{section}.enabled

Step 4: Update Topic Modal to Fetch from CMS


// In {Pillar}TopicModal.jsx - fetchProducts function

const fetchProducts = async () => {
    // FIRST: Try admin-assigned products
    const res = await fetch(`${API_URL}/api/{pillar}/topic-products/${topicSlug}`);
    if (res.ok) {
        const data = await res.json();
        if (data.products?.length > 0) {
            setProducts(data.products);
            return;
        }
    }
    // FALLBACK: Default products
    ...
};
