What is the Page CMS?

The Page CMS is a centralized, database-driven content management system that allows admins to control every aspect of all 14 pillar pages from the Admin Panel without writing code.

✔ What it does
  • - NO hardcoded content
  • - Admins control everything
  • - Consistent 14-pillar structure
  • - Dynamic page rendering
  • - Personalization preserved
🛠 Key Benefits
  • - Single source of truth
  • - Reusable components
  • - Scalable architecture
  • - Easy content updates
  • - No code deployments needed

The Generic CMS Component

PillarPageCMS.jsx is a single, reusable component that powers the admin CMS UI for 12 of the 14 pillars. Learn and Paperwork have custom components due to their unique requirements. 

import PillarPageCMS from '../components/admin/PillarPageCMS';

// In Admin.jsx render:
{activeTab === 'care-cms' && <PillarPageCMS pillar="care" />}
{activeTab === 'fit-cms' && <PillarPageCMS pillar="fit" />}
// ... etc for all 12 generic pillars

API Endpoints

Method Endpoint Description
GET /api/{pillar}/page-config Get page configuration for any pillar
POST /api/{pillar}/page-config Save page configuration for any pillar
⚠ Valid Pillars for Generic Endpoint
care, fit, travel, stay, dine, enjoy, celebrate, emergency, advisory, farewell, adopt, shop
📋 Dedicated Endpoints
  • /api/learn/page-config - Learn page (has additional fields)

How to Make a Page CMS-Driven

1

Verify CMS Admin Exists

Login as admin and check if the CMS tab exists at /admin?tab={pillar}-cms

2

Add CMS State to Page Component

// At top of component (e.g., CarePage.jsx):
const [cmsConfig, setCmsConfig] = useState({
  title: 'Default title with {petName}',
  subtitle: 'Default subtitle',
  askMira: { enabled: true, placeholder: 'Search...' },
  sections: { askMira: { enabled: true }, ... }
});
const [cmsCategories, setCmsCategories] = useState([]);
const [cmsConciergeServices, setCmsConciergeServices] = useState([]);
3

Add fetchCMSConfig Function

const fetchCMSConfig = async () => {
  try {
    const response = await fetch(`${API_URL}/api/{pillar}/page-config`);
    if (response.ok) {
      const data = await response.json();
      if (data.config) setCmsConfig(prev => ({ ...prev, ...data.config }));
      if (data.categories?.length) setCmsCategories(data.categories);
    }
  } catch (error) { /* Fallback to defaults */ }
};
4

Call in useEffect

useEffect(() => {
  fetchCMSConfig();  // Load CMS config
  fetchOtherData();  // Other fetches
}, []);
5

Create Computed Values with Fallbacks

const categories = cmsCategories.length > 0 ? cmsCategories : DEFAULT_CATEGORIES;
const pageTitle = cmsConfig.title?.replace('{petName}', activePet?.name || 'your pet');
6

Render Sections Conditionally

{cmsConfig.sections?.askMira?.enabled !== false && (
  <AskMiraSection 
    placeholder={cmsConfig.askMira?.placeholder} 
  />
)}
