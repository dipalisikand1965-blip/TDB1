Dine Pillar — Complete Specification (Session 39, Mar 2026)

The /dine page is the food and nutrition operating centre for each pet. It was redesigned following the /celebrate-soul template — warmer, more protective, food-first. Built with two switchable tabs: Eat & Nourish and Dine Out.

"On /celebrate, Mira is joyful and excited. On /dine, Mira is protective, knowledgeable, quietly confident — a nutritionist who loves this dog."

Page Architecture (Top to Bottom)

  1. Hero — Pet front and centre, Mira's food-first voice, soul chips (allergy, favourite food, personality)
  2. Two Main Tabs — "Eat & Nourish" | "Dine Out"
  3. Tummy Profile — Collapsed bar (default), expands to 4-cell grid: Loves · Allergic To · Nutrition Goal · Health Note
  4. Five Dine Dimensions — Daily Meals, Treats & Rewards, Supplements, Frozen & Fresh, Homemade & Recipes
  5. Mira's Meal Pick — Single curated meal recommendation, personalised to Tummy Profile
  6. Guided Nutrition Paths — 3 of 6 paths surfaced by Mira (allergy, treatment, fresh food, senior, puppy, weight)
  7. Dine Out Tab — Pet-friendly restaurants via Google Places API, filtered by Mira for breed suitability
  8. Dining Concierge Services — 4 illustrated cards, below both tabs (Restaurant Discovery, Reservation, Etiquette, Venue Advisory)

Data Protection — locally_edited Flag

Shopify sync runs on every deployment. Products with locally_edited: true are protected — Shopify sync only updates available and shopify_updated_at, never overwrites admin changes. All admin product creates/edits set this flag automatically.

CONSOLIDATE DATA Button

Mobile Pet Dashboard — Scrambled UI
The member dashboard is not responsive on mobile. Grid layout breaks. Needs mobile-first CSS fix.
P3
AI Image Generation — Non-Persistent Background Tasks

P3
Membership Tier Gating — Not Enforced
Silver/Gold/Platinum tiers are in the DB but tier-based access restrictions and upgrade prompts are not fully implemented.
