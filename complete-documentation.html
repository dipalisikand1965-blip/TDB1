Soul Product Pipeline

The pipeline flows ONE WAY: breed_products → (on generation) → products_master → Frontend (filtered by breed_tags + pillar)

Step by Step

StepWhat happensKey fields
1. SeedProduct entry created in breed_products with promptid, breed, pillar, product_type, mockup_prompt, is_active=true, mockup_url=null
2. GeneratePOST /api/mockups/generate-batch triggers AI image generation (GPT-Image-1)mockup_url gets set, is_mockup=true
3. Sync_sync_to_products_master() called automatically → upserts into products_master by idcloudinary_url, breed_tags, product_type, visibility={status:active}, soul_made=true
4. Member seesFrontend fetches /api/admin/pillar-products?pillar=X&breed=YFiltered by pillar_products_routes.py → _should_show_for_breed()
Known gap: If products are seeded but images already exist from a previous run, the batch generator SKIPS them (pending = no mockup_url check). The sync never re-runs for those products. Fix: After any batch seeding where products already have images, manually run the product_type sync script.

Sync Fix Script (run after batch seeding)

python3 << 'PYEOF'
from pymongo import MongoClient
from datetime import datetime, timezone
client = MongoClient('mongodb://localhost:27017')
db = client['pet-os-live-test_database']
now = datetime.now(timezone.utc).isoformat()

for bp in db.breed_products.find({'is_active': True}, {'_id': 0}):
    pid = bp.get('id')
    if not pid: continue
    pm = db.products_master.find_one({'id': pid}, {'_id': 0, 'product_type': 1, 'visibility': 1})
    if not pm: continue
    correct_type = bp.get('product_type') or bp.get('category') or 'soul_made'
    img = bp.get('cloudinary_url') or bp.get('mockup_url')
    update = {'product_type': correct_type, 'soul_made': True, 'updated_at': now}
    if img: update.update({'cloudinary_url': img, 'image_url': img, 'image': img})
    if not pm.get('visibility'): update['visibility'] = {'status': 'active'}
    db.products_master.update_one({'id': pid}, {'$set': update})
print('Sync complete')

Database Schema

products_master (Member-facing catalog)

{
  id:              "breed-labrador-bowl",       // unique
  name:            "Labrador Ceramic Bowl",
  pillar:          "dine",
  product_type:    "bowl",                      // NOT 'soul_made' — fix if stuck
  soul_made:       true,
  is_active:       true,
  visibility:      { status: "active" },        // REQUIRED — must exist
  breed_tags:      ["labrador"],                // array, lowercase, underscores
  cloudinary_url:  "https://res.cloudinary.com/duoapcx1p/...",
  image_url:       "same as cloudinary_url",
  ai_generation_locked: true/false,            // true = AI batch will skip this product
  mockup_url:      "same as cloudinary_url",
  original_price:  0,
  tags:            ["labrador", "dine", "bowl", "soul_made"]
}

breed_products (Soul Generator source of truth)

{
  id:              "breed-labrador-bowl",
  breed:           "labrador",                  // lowercase, underscore slug
  pillar:          "dine",
  product_type:    "bowl",
  mockup_prompt:   "Professional product photography...",
  mockup_url:      "https://res.cloudinary.com/...",  // null until generated
  cloudinary_url:  "same",
  is_active:       true,
  is_mockup:       true/false,                 // true after generation
  soul_made:       true,
  breed_tags:      ["labrador"]
}
