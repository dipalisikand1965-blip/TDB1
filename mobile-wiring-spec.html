6. Implementation Patterns

Product Fetch + Intelligence Filter

// ALWAYS use this pattern in every mobile page
useEffect(() => {
  if (!currentPet?.id) return;
  fetch(`${API_URL}/api/admin/pillar-products?pillar=care&limit=200`, {
    headers: token ? { Authorization: `Bearer ${token}` } : {}
  })
    .then(r => r.ok ? r.json() : null)
    .then(d => {
      if (d?.products) {
        const filtered = filterBreedProducts(d.products, currentPet?.breed);
        setAllRaw(filtered);
      }
    })
    .catch(() => {});
}, [currentPet?.id, token]);

// Then in render:
const allergies = getAllergies(currentPet);
const intelligent = applyMiraIntelligence(allRaw, allergies);
const subCats = ['All', ...new Set(intelligent.map(p => p.sub_category).filter(Boolean))];
const products = subCat === 'All' ? intelligent : intelligent.filter(p => p.sub_category === subCat);

Frontend Flow

  1. User taps "Get Help" button
  2. tdc.book({ urgency: 'critical' }) fires
  3. POST /api/notifications/emergency-whatsapp fires simultaneously
