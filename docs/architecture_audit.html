🎁 Bundle Architecture

There are two bundle systems in the application. The unified system (recommended) is the source of truth for all pricing and cross-pillar operations.

✓ Unified Bundle System (Recommended)
  • • Collection: bundles
  • • 39 bundles across all 13 pillars
  • • Managed by: BundlesManager.jsx
  • • Priced via: PricingHub → Pillar Bundles tab
  • • Fields: id, name, pillar, original_price, bundle_price, discount, active, items, image_url
⚠️ Legacy Pillar-Specific Bundle System
  • • Collections: {pillar}_bundles (6+ collections)
  • • Managed by: Individual Pillar Managers
  • • NOT shown in PricingHub (use unified instead)
  • • Some have inconsistent item schemas (strings vs objects)

🔧 Fixes Applied
  • ✓ Fixed
    Product Activation — 3,960 products in products_master had no active field. Script ran: update_many(active: missing → set active: True). Now 3,987 products are active and will display in the app.
  • ✓ Fixed
    PricingHub Pillar Bundles Tab — Was fetching from 5 pillar-specific endpoints with wrong field names. Rewritten to use GET /api/bundles?active_only=false (all 39 bundles), grouped by all 13 pillars, with correct original_price/bundle_price fields and inline price editing.
  • ✓ Fixed
    Bundle Pricing PATCH Endpoint — Added PATCH /api/bundles/{id}/pricing for lightweight pricing updates (original_price, bundle_price, active). Automatically recalculates discount %.
  • ✓ Fixed
    BundlesManager Crash — Bundle items in bundles collection can be objects {name, price}. Added typeof item === 'object' ? item.name : item check to prevent "Objects are not valid as React child" crash.
  • ✓ Fixed
    LearnManager Bundle Crash — Same typeof check applied to LearnManager's bundle items rendering.
  • ✓ Fixed
    EnjoyManager Crash — credentials variable was not defined in scope when passed to PillarBundlesTab. Removed the undefined prop reference.
  • ✓ Fixed
    Notification Sort Bug — Member notifications were sorting by created_at which had mixed types (BSON datetime + strings). Fixed by sorting by _id (always monotonically increasing).
  • ✓ Added
    Services Tab Added — Added PillarServicesTab to 7 pillar admin pages: FarewellManager, EmergencyManager, AdoptManager, FitManager, LearnManager, PaperworkManager, AdvisoryManager.
  • ✓ Added
    Service Pricing in PricingHub — Added Service Pricing tab to PricingHub.jsx with new backend endpoint PATCH /api/service-box/services/{id}/pricing.
  • ✓ Fixed Mar 15
    Services Architecture — Shop Removal + Pillar Assignment — 561 services with pillar: '' assigned to correct pillars using keyword heuristics. 392 services with pillar: 'shop': product-type entries deactivated, real services reassigned. ServiceBox admin now shows only 13 true service pillars (no 'shop'). Shop page is for products only.
  • ✓ Fixed Mar 15
    Celebrate Products Admin — Full CRUD with 1,499 Products — GET /api/celebrate/admin/products merged from celebrate_products (4 manual) + products_master (1,495 Shopify). Shows 1,499 products with full CRUD, pagination, search, category filter, image upload button, and AI image generation button.
⚠️ Remaining Gaps & Next Steps
  • P1 Pending
    561 Services have no pillar assignment — These services sit in services_master with pillar: null. Cannot be filtered by pillar in admin. Need to manually assign or bulk-assign pillars based on service category/type.
  • P1 Pending
    392 Services have pillar="shop" — These are likely generic shop services. They need either a real pillar assignment or a "shop" pillar to be created in the UI.
  • P2 Pending
    Consolidate Legacy Pillar Bundle Collections — 6 pillar-specific bundle collections (celebrate_bundles, dine_bundles, etc.) are separate from the unified bundles collection. Consider migrating all to the unified collection or at least syncing them.
  • P2 Pending
    care_bundles items are objects — Items in care_bundles are {name: string, price: number} objects. The CareManager doesn't render them directly so no crash, but editing bundles will show [object Object] in text fields. Need to normalize.
  • P3 Pending
    Razorpay Checkout Failure — Known bug, not yet investigated.
  • P3 Pending
    Love Memory Drawer — Feature request awaiting specifications.
