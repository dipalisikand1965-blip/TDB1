Mira AI — Product Intelligence

Mira Picks

AI-scored product recommendations using Claude Sonnet. Scored per-pet per-pillar. Falls back to "Mira Imagines" cards when no scores available.

Claude Sonnet 4.5 Auto-trigger on page load Promise.allSettled (5s timeout)

Mira Score Engine

POST /api/mira/score-for-pet triggers scoring for all products/services in a pillar for a given pet. Stored in pet_soul_scores.

TDC CTA Instructions

CTA Chain (from TDC_CTA_Instructions.docx)

Every "Book via Concierge" or "Find & Book" button across all pillar pages must wire into the full service desk chain:

StepComponent/ActionResult
1User clicks "Book" on any service/product/find cardConciergeToast shown (bottom notification)
2ConciergeToast confirms booking intentAPI call to /api/concierge/{pillar}-service
3Backend creates concierge_intakeservice_desk_ticket created
4admin_notification createdAdmin inbox shows new ticket
5User sees confirmation"Mira is on it" message + ticket ID
Find & Book CTAs: PlayNearMe "Book this spot", CareNearMe "Book care", GoSoulPage "Find a Stay" → all should trigger handleNearMeBook() which creates a ticket and shows ConciergeToast. Currently these are placeholder buttons — wire up is P1.
