# Mira OS — Complete Product Doctrine
## The Pet Operating System
### Last Updated: February 13, 2026

---

## 1. What We Are Building

**Mira OS is the world's first Pet Operating System**

- Not an app.
- Not a chatbot.
- Not a marketplace.
- Not a recommendation engine.

**It is an intelligent operating system that knows each pet deeply and orchestrates their life.**

### Core Definition
We are building an identity-first intelligence system that:

```
KNOWS the pet → UNDERSTANDS the pet → GUIDES decisions → EXECUTES actions
```

Everything flows from this order.

---

## The 5 Core Principles

### 1. Pet First. Always.
- Everything starts from the pet identity.
- Nothing is generic.
- Nothing assumes from breed.
- Parent preferences are secondary to pet safety.
- If the pet is not understood → system must ask.

### 2. Identity Before Action
First know → then recommend → then execute.

```
MOJO (Identity)
      ↓
   Context
      ↓
   Decision
      ↓
  Execution
```

Without identity, Mira is just a chat tool.

### 3. Memory is the Product
Memory is not a feature. **Memory = the product.**
- Soul form answers
- Health data
- Behaviour patterns
- Interaction history
- Decisions made
- Services used
- What worked / didn't work

Every interaction improves the system.

### 4. Concierge Intelligence, Not Search Results
We never return lists. We return:
- filtered choices
- curated actions
- personalized reasoning
- execution options

### 5. Execution, Not Information
Information alone is incomplete.

Mira must always move toward:
- arrange
- schedule
- coordinate
- confirm
- execute

## 4. The System Architecture (How Everything Fits)

Mira OS has three layers.

### Layer 1 — Identity (WHO the pet is)
**MOJO**
The pet profile. Contains:
- personality
- health
- behaviour
- diet
- routine
- documents
- history
- preferences

**This is the core database.**

### Layer 2 — Intelligence (WHAT to do)
- **PICKS** - Personalized recommendations.
- **INSIGHTS** - Patterns and learning.
- **TODAY** - Time-based needs.

**System thinking lives here.**

### Layer 3 — Execution (MAKE it happen)
- **SERVICES** - Task orchestration.
- **CONCIERGE** - Human escalation.

**Action happens here.**

### Layer 4 — Knowledge (UNDERSTAND better)
- **LEARN** - Education and guidance.

### Navigation Philosophy
```
MOJO → WHO the pet is
TODAY → What needs attention now
PICKS → What fits the pet best
SERVICES → Make it happen
INSIGHTS → What we learned
LEARN → Understand better
CONCIERGE → Human help
```

# Mira OS Tabs Definition (What each area is)

## 1) MOJO

### What it is
Pet Identity Layer. The single source of truth about the pet.

### Contains
- Pet snapshot: name, breed, age band, sex, weight, coat type, location
- Soul answers (55+) + completeness %
- Derived traits with confidence (likes/dislikes, anxiety triggers, chew style, diet type)
- Health vault: allergies, vaccination, vet info, documents
- Routine basics: walk frequency, sleep pattern (if captured)
- Documents vault + timeline (lazy-loaded)

### Never contains
- Fulfilment actions / booking forms
- "Buy now" behaviour
- Random suggestions not tied to stored data

### Output it powers
- What Mira can say without asking
- Safety filters
- Picks ranking
- Today reminders accuracy

---

## 2) TODAY

### What it is
Time Layer. Everything that matters "now / soon" across all pets and all pillars.

### Contains (cards)
- Due/overdue: grooming cadence, parasite dose, vaccines, vet follow-ups
- Upcoming: birthday countdown, travel date, renewals
- Risk alerts: heatwave walk safety, tick season, fireworks anxiety window
- Expiring documents (rabies certificate, travel docs)
- Active task deadlines: "Awaiting your confirmation" / "Payment needed"
- "Other pets need attention" (collapsed section)

### Never contains
- Shopping/catalog browsing
- Long educational articles
- Deep analytics charts (those go to Insights)

### One-tap actions
Each Today card must have a clear action:
- Arrange / Book / Schedule / Upload / Confirm / Remind me

### Where it comes from
- Memory (cadences + health flags + calendar)
- Tasks system (statuses)
- Weather/location signals (if available)

---

## 3) PICKS

### What it is
Intelligence Layer. Live recommendations that update every chat turn.

### Core doctrine
**Catalogue-first. Concierge-always.**
Picks must never be empty.

### Contains
Each refresh shows 6–10 items:
- 4–6 picks from the Active Pillar
- Minimum 1 service card
- 1–2 essentials only if relevant (avoid repetition)
- Emergency/Farewell override: actions/services > products

### Two card types only

**A) Catalogue Pick Card (if found)**
- Uses catalogue object rendering
- No price unless stored

**B) Concierge Pick Card (if not found)**
- Brand-free title
- Why it fits (personalised to pet traits)
- Selection rules (bullets)
- Safety line (1 line)
- Fulfilment: "Arranged for {Pet}" (not "concierge will source")
- CTA: Request / Arrange / Plan / Start

### Never contains
- Marketplace inventory feel
- Random lists without pet-based reasoning
- Medical directives or dosing

### Relationship to 14 pillars
Picks always has an Active Pillar (and optional secondary pillar).
When conversation changes pillar, Picks switches immediately.

---

## 4) SERVICES

### What it is
Execution Layer. Where coordination happens.
**Services is not a pillar. Every service task belongs to one of the 14 pillars.**

### Contains
- Service "cards" (arranged-for tone) that create tasks
- Task inbox: requested / in progress / awaiting user / confirmed / completed
- Forms that ask only missing, moment-specific details
- Order tracking + subscriptions + refills (because operational)

### Examples (cross-pillar)
- Grooming booking (Care)
- Vet appointment coordination (Care/Advisory)
- Boarding/daycare (Stay)
- Pet-friendly hotel shortlist + booking (Stay/Travel)
- Party setup + photographer (Celebrate)
- Trainer matching + session booking (Learn)
- Diet transition plan setup (Dine)
- Emergency transport coordination (Emergency)
- Document handling (Paperwork)

### Never contains
- Deep education guides (Learn)
- Long-term charts (Insights)
- Open-ended chat (chat stays in chat)

### Required behaviours
- Creating a task never "teleports" user away from chat
- Confirm action shows undo toast (5 seconds)
- Any notification deep-links to the task detail screen

---

## 5) INSIGHTS

### What it is
Pattern Layer. Longitudinal intelligence about the pet over time.

### Contains
- Trend cards: weight, activity, grooming cadence adherence
- Risk scores: dental risk, seasonal parasite exposure, anxiety triggers (non-diagnostic)
- "What's changed" summaries month over month
- Spending patterns (if you track)
- "Works / didn't work" learning from completed tasks and feedback

### Never contains
- Booking forms
- Product browsing grids
- Time-critical reminders (those live in Today)

### Output it powers
- Better future Picks
- Proactive Today items
- Smarter questions (ask less over time)

---

## 6) LEARN

### What it is
Knowledge Layer. Calm, structured education.

### Contains
- Bite-sized guides: nutrition basics, coat care, travel prep, enrichment ideas
- Breed-specific overviews (as education, not assumptions about the pet)
- Emergency first aid basics (non-medical, "call vet" emphasis)
- Puppy starter guide / senior guide

### Never contains
- Booking/fulfilment actions
- Selling language
- Medical diagnosis, dosage, or prescriptions

### CTA style
- "Learn more" / "Save this" / "Send to my concierge" (optional handoff)

---

## 7) CONCIERGE

### What it is
Human escalation layer. When the user wants a person or the system needs human judgement.

### Contains
- WhatsApp / Email contact
- Escalate request (attach context pack + picks + task summary)
- Upload document to team
- Track fulfilment tasks
- Feedback + technical help

### Never contains
- Regular browsing
- Primary execution forms (those are Services)
- Generic help articles (those are Learn)

### Emergency behaviour
Emergency starts in TODAY (red card/top), but can escalate to Concierge instantly.

---

## How the 14 Pillars fit into this OS (important)

**The 14 pillars are classification + routing, not navigation tabs.**

Pillar lives on:
- every chat intent
- every pick
- every task
- every service card
- every insight category

But users navigate by OS layers:
```
Who (MOJO) → Now (TODAY) → Suggest (PICKS) → Do (SERVICES) → Understand (INSIGHTS) → Learn (LEARN) → Escalate (CONCIERGE)
```

### One line that removes confusion
**Tabs are OS layers. Pillars are the taxonomy underneath. Services executes tasks across pillars; Picks recommends across pillars; Today surfaces time-based items across pillars.**
