## Concierge Card Doctrine

### Pet First - Always

The Concierge Card is **personalized to the PET**, not the human.

**Hierarchy:**
1. **Pet Name** - "Designed for Mystique"
2. **Soul Traits** - "who needs calm environments"
3. **Breed** (fallback) - "Shih Tzu" (only if soul data insufficient)

### Card Structure

```
┌────────────────────────────────────────┐
│  ✨ CONCIERGE PICK                     │
│                                        │
│  🎂 Custom Birthday Party              │
│                                        │
│  Designed for Mystique                 │
│  who needs calm environments           │
│                                        │
│  ┌──────────────────────────────────┐  │
│  │  💬 Let Mira Arrange This        │  │
│  └──────────────────────────────────┘  │
│                                        │
│  Your Pet Concierge® will contact you  │
│  within 2 hours                        │
└────────────────────────────────────────┘
```

### Where It Appears

Every pillar page must have:
1. **Product Cards** (if products match pet profile) → Cart flow
2. **Concierge Cards** (always) → Ticket flow

**If no products match**: Concierge Cards take the hero position.

## Notification Doctrine

### For Products (Transactional)
- Email: Receipt with order details
- In-App: "Order #123 confirmed for Mystique"

### For Concierge Requests (Consultative)
- Email: "We're on it for Mystique! Your Pet Concierge® will contact you."
- In-App: "Concierge request submitted" + thread link
- Follow-up: Real-time updates in Services tab
