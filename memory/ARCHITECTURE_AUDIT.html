3. Critical Gaps & Issues

🔴 CRITICAL — Only 28 of 4,334 Products are Active

!
Product Box: 4,334 total but only 28 are active: true
All 4,306 remaining products have active: false or default value. These products will not show on the frontend pillar pages. This needs a bulk activation (admin can set active=true per pillar via Product Box filters).

🔴 CRITICAL — 3 Orphan Product Collections Not in Product Box

!
products (1,379 items) — Soul Products admin uses this. NOT in Product Box.
Soul Products (Soul Made, Soul Selected, Soul Gifted) live in the products collection. They are NOT in unified_products (Product Box). They have no pillar assignment. They need to be migrated to Product Box with pillar tags.
!
products_master (3,987 items) — Orphan. What is this?
This collection appears to be a legacy mirror of products. Unclear if this is the source of truth or a cache. Needs investigation and possible consolidation.
!
breed_products (2,569 items) — Breed-specific products. NOT in Product Box.
Breed-specific products are in a separate collection and not accessible from the Product Box. These need to be tagged to their relevant pillar (care, fit, dine etc.) and added to unified_products.

🔴 CRITICAL — 561 Services Have No Pillar Assignment

!
561 of 1,118 services have pillar: "" or empty array
These services exist in the Service Box but are not visible in any pillar admin tab. The pillar Services tab shows 0 for these services. They need to be assigned to a pillar via bulk update or individual edit in Service Box.

🔴 CRITICAL — Bundles are Fragmented Across 14+ Collections

!
14 separate {pillar}_bundles collections + 1 main bundles collection
Every pillar has its own bundles collection. The main bundles collection only has 39 items. There is NO single "Mother Bundle" collection. This makes pricing sync, bulk operations, and data consistency impossible.
Collections: care_bundles, adopt_bundles, fit_bundles, learn_bundles, stay_bundles, celebrate_bundles, farewell_bundles, paperwork_bundles, product_bundles, emergency_bundles, advisory_bundles, dine_bundles, travel_bundles, enjoy_bundles

🟡 MEDIUM — Bundles Not in Pricing Hub

!
Pricing Hub has Product Pricing + Service Pricing tabs but NO Bundle Pricing tab
Bundle prices cannot be bulk-edited from the centralized Pricing Hub. Admin must edit each bundle individually in the pillar manager.

🟡 MEDIUM — Celebrate Subcategories Not Visible as Pillar Sub-sections

!
Celebrate has 8+ subcategories (Pupcakes, Treats, Desi Treats, etc.) but these are NOT structured as named sub-sections in Product Box pillar view
The 1,263 celebrate products have category tags, but the Product Box pillar view does not break them down by subcategory. Admin cannot see "Celebrate → Pupcakes (42 products)" etc.

4. Intended Architecture (Target State)

MASTER RECORD: Product Box (unified_products)

Every product from every source (Soul Products, breed-specific, pillar-specific, imported) MUST live here. Fields required: id, name, primary_pillar, pillars[], category, subcategory, price, active, image_url, product_type (soul_made/standard/breed)

↓ feeds into ↓

PILLAR PRODUCT TABS (14 pillars)

Each pillar's admin Products tab filters unified_products by primary_pillar="{pillar}". Admin can see products sub-category wise and activate/deactivate/edit/generate AI images per product from here.

PRICING HUB — Product Pricing Tab

Reads from unified_products. Allows bulk price editing across all pillars. Filters by pillar.

MASTER RECORD: Service Box (services_master)

Every service from every pillar MUST live here. Fields required: id, name, pillar (string, not array), base_price, discounted_price, active, image_url, is_bookable, duration_minutes

↓ feeds into ↓

PILLAR SERVICES TABS (14 pillars)

Each pillar's Services tab filters services_master by pillar="{pillar}". Active/Inactive toggle available per service.

PRICING HUB — Service Pricing Tab ✅ (Added Mar 15)

Reads from services_master. Allows bulk price editing. PATCH endpoint: /api/service-box/services/{id}/pricing

MASTER RECORD: Bundles (bundles — NEEDS CONSOLIDATION)

All pillar bundles MUST be in ONE collection. Required fields: id, name, pillar (string), price, original_price, items[] (strings), active, image_url, watercolor_image

⚠️ Currently: 14 separate pillar collections must be MERGED into 1

↓ feeds into ↓

PILLAR BUNDLES TABS (14 pillars)

Filter bundles by pillar="{pillar}". Admin can create/edit/delete/AI-generate images per bundle.

PRICING HUB — Bundle Pricing Tab ❌ (MISSING)

Needs to be added. Should read from consolidated bundles collection with pillar filter and inline price editing.

7. Recommended Fix Plan

🔴 Priority 1 — Fix Active Status (Quick Win)

Bulk-set active: true for products that should be visible. Current state: only 28/4334 are active. Use the Product Box admin → filter by pillar → bulk activate. Or run a one-time migration script.

Fix: Add "Bulk Activate by Pillar" button in Product Box admin, or run: db.unified_products.updateMany({active: {$ne: true}}, {$set: {active: true}})

🔴 Priority 2 — Assign Pillar to 561 Unclassified Services

561 services have no pillar. Use Service Box admin → filter by pillar="" → bulk assign pillar. Or add a "Pillar Bulk Assignment" tool in Service Box.

🔴 Priority 3 — Consolidate 14 Bundle Collections → 1

Write a migration script that reads all {pillar}_bundles collections, normalizes the items[] field (convert objects to strings), ensures all have a pillar field, and inserts into the main bundles collection. Then add a Bundle Pricing tab to Pricing Hub.

🟡 Priority 4 — Add Soul Products to Product Box

Migrate products collection (1,379 Soul Products) to unified_products with product_type: "soul_made" and appropriate primary_pillar tags. Deprecate the separate products collection for the Product Box view.

🟡 Priority 5 — Add Bundle Pricing Tab to Pricing Hub

After bundle consolidation, add a "Bundle Pricing" tab to PricingHub.jsx. Use GET /api/bundles?limit=200 endpoint with pillar filter and inline price editing via PATCH /api/bundles/{id}/pricing.
