🚨 Critical Notes for Next Agent — READ BEFORE ANYTHING ELSE

DO NOT BREAK THE PRODUCT BOX. The products_master MongoDB collection is the Single Source of Truth (SSOT) for ALL products across ALL pillars. Never create a separate products collection for any pillar. Always query/modify products_master.
DO NOT OVERWRITE SEEDED DATA ON DEPLOYMENT. Products (DM-XXX, TR-XXX, SP-XXX, FF-XXX, HR-XXX), services, and bundles are seeded in the database. No seed scripts should run on every server startup. Always check DB before seeding.
IMAGE HISTORY WARNING. In a previous session, an agent added a faulty image filter that broke all product images by blocking static.prod-images.emergentagent.com URLs. This was fixed. DO NOT add any image URL filtering logic to resolveEntityImage() or resolvePickImage() functions. They should simply return the first valid URL starting with "http".
REACT STATE BATCHING BUG — Already Fixed. ProductBoxEditor.jsx was calling multiple updateField() in sequence inside an async XHR callback. Due to React 18 batching, only the last call persisted. Fixed by creating one newProduct object with all fields set, then calling setProduct(newProduct) once. DO NOT revert this pattern.
ADMIN AUTH PATTERN. The admin panel uses HTTP Basic Auth (username/password). Admin XHR calls must use Authorization: Basic base64(user:pass), NOT Bearer tokens. Use the getAdminAuth() helper already present in ProductBoxEditor, BundlesManager, CelebrateManager, ServiceBox. Check BOTH localStorage.adminAuth AND sessionStorage.admin_auth.
PORTAL Z-INDEX PATTERN. All modals on pillar pages use createPortal(content, document.body) with zIndex: 10002. The sticky navbar is at z-[10000]. Never set a modal below 10002 or it will render behind the navbar.

Project Overview & Vision

The Doggy Company® — Mission

A Pet Life Operating System powered by Mira AI. Everything a pet parent needs across 14 life pillars — from food and celebration to healthcare and farewell — all deeply personalized to their specific dog's soul profile, breed, health, loves, and allergies.

URL: https://health-vault-loop.preview.emergentagent.com

Core Philosophy

  • "Mira" is the AI brain — knows every pet deeply
  • Everything is ultra-personalized to pet's soul profile
  • All pillars follow the same golden pattern: Hero → Category Strip → TabBar → Dimensions → Mira Picks → Concierge
  • Products/services served from one SSOT: products_master
  • Recommendations powered by Mira Score Engine (Claude Sonnet)

Tech Stack

  • Frontend: React 18, Tailwind CSS, Framer Motion, Lucide Icons
  • Backend: FastAPI (Python), MongoDB (Motor async), Cloudinary
  • AI: Claude Sonnet 4.6 (scoring/chat), OpenAI GPT-Image (images)
  • Integrations: Google Places API, Shopify (import), Razorpay (payments)
  • Auth: JWT for users, HTTP Basic for admin

Product Box Workflow (How to Add/Edit Dine Products)

  1. Go to /admin → COMMERCE → Product Box
  2. Click "Dine" pill filter (shows ~789 dine products)
  3. Filter further by Category (e.g., "Daily Meals") using the category dropdown
  4. Click any product → opens 6-tab ProductBoxEditor
  5. Go to Pillars tab → assign Category, Sub-Category, Status
  6. Go to Media tab → generate AI image or upload photo
  7. Save → product immediately appears on /dine front-end
